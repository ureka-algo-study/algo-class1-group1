## ⚙️ 운영 방식

```
각자 브랜치에서 작업 후 Main으로 PR 날리기
```

1. `main` 브랜치에서 **본인 이름으로 브랜치** 생성
   ```bash
   git checkout -b 김가원
   ```
2. 본인 폴더 안에 풀이 파일 추가
3. 커밋 & 푸시 후 **`main`으로 PR** 생성
4. 최소 1명 이상의 **Approve** 후 머지

## 📋 규칙

| 규칙 | 내용 |
|------|------|
| 작업 범위 | **본인 폴더 내에서만 작업** (타인 폴더 수정 금지) |
| 사용 언어 | **자유** (Python, Java, C++, JavaScript 등 제한 없음) |
| 머지 조건 | **최소 1명 이상의 Approve** 후 머지 |
| PR 양식 | 제공된 PR 템플릿 성실히 작성 |
| 리뷰 문화 | 건설적인 피드백, 질문 환영 |
